What is Secure Cluster Connectivity?

Secure Cluster Connectivity (SCC), also known as No Public IP (NPIP), is an Azure Databricks deployment model that eliminates public IP addresses from cluster nodes in the compute plane.

When SCC is enabled, cluster nodes have no public IP addresses. Instead, clusters initiate outbound connections to the Databricks control plane via a secure relay on port 443 (HTTPS). The control plane never initiates inbound connections to the compute plane — all communication flows outbound from the customer's network.

All traffic stays on the Microsoft Azure network backbone, meaning data never traverses the public internet. This architecture dramatically reduces the attack surface while simplifying network security rules.

Starting with API version 2024-05-01 and later, Secure Cluster Connectivity is the default for all new Azure Databricks workspaces.

Key Concepts

Control Plane

Managed by Databricks in the Databricks-managed subscription. The control plane hosts the web application, REST API, cluster management, and job scheduling services. Customers access it via the workspace URL.

Classic Compute Plane

Runs in the customer's Azure subscription (or a managed VNet). The compute plane contains the actual Spark clusters (VMs) and DBFS storage. With VNet injection, clusters run inside the customer's own virtual network.

Secure Relay

The SCC relay is the tunnel between the compute and control planes. Clusters initiate outbound HTTPS (port 443) connections to the relay endpoint. The control plane sends management commands through this tunnel. No inbound ports are needed on the compute side.

How SCC Works

  1. Workspace is created with enableNoPublicIp = true
  2. Cluster node VMs are provisioned without public IP addresses
  3. Each cluster initiates an outbound HTTPS connection to the SCC relay endpoint
  4. The control plane sends management commands through this secure tunnel
  5. All network traffic traverses the Microsoft Azure backbone — never the public internet

With Secure Cluster Connectivity enabled, there are no open inbound ports on your compute nodes and no public IP addresses. This significantly reduces your attack surface.
