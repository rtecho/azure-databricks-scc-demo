Why SCC

Why Secure Cluster Connectivity?

Eliminate public IP exposure, reduce attack surface, and meet enterprise compliance requirements

Security Benefits

No Public IP Addresses

Cluster VMs have zero public IPs. Attackers cannot discover or target your compute nodes from the internet.

No Inbound Ports

No open inbound ports (22, 5557) on your subnets. The control plane communicates via an outbound-only secure relay.

Azure Backbone Only

All traffic between control plane and compute plane stays on the Microsoft network backbone — never traverses the public internet.

Reduced Attack Surface

Fewer network entry points = fewer exploitation opportunities. SCC is the most restrictive default network posture for Databricks.

Compliance & Governance

SCC directly supports controls across major compliance frameworks by eliminating public network exposure on compute infrastructure.

Framework Requirement How SCC Helps
CIS Azure Benchmark Restrict public IP assignment No public IPs on cluster nodes
NIST 800-53 SC-7 Boundary Protection Eliminates inbound network paths to compute
PCI DSS Restrict inbound traffic No inbound ports open on compute subnets
ISO 27001 A.13.1 Network security All traffic on Azure backbone, no public exposure
SOC 2 CC6.1 Logical access security Outbound-only relay reduces unauthorized access risk

Operational Benefits

Default for New Workspaces

Starting with API version 2024-05-01, enableNoPublicIp defaults to true. New workspaces get SCC automatically.

No Extra Cost

SCC itself has no additional licensing cost. The NAT Gateway for VNet injection egress is the only added infrastructure cost.

Seamless Upgrade Path

Existing workspaces can enable SCC by setting enableNoPublicIp to true via portal, ARM template, or Terraform (provider 3.41.0+). Network update takes ~15 minutes.

SCC Is Not the Same as Private Link

SCC prevents public IPs on compute nodes but workspace access (web UI, APIs) can still be public. For full private isolation, combine SCC + VNet Injection + Private Link (front-end and back-end).
